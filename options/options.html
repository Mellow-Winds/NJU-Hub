关于插件

NJU Hub — 南大智慧终端 v2.0.1

AI 配置教程

1. 获取 API Key — 前往大模型服务提供商的开放平台注册并获取 API Key。常用提供商包括 SiliconCloud、OpenAI、智谱 AI 等。注册后在控制台创建 API Key 并复制。

2. 配置 API Base URL — 将提供商提供的 API 地址填入对应模块的「API Base URL」输入框。通常格式为 https://api.xxx.com/v1，具体以提供商文档为准。

3. 填写 API Key — 将步骤 1 获取的 Key 粘贴到「API Key」输入框中，该 Key 仅保存在你本地浏览器的插件存储中。

4. 选择模型 — 在「Model」输入框中填写模型名称（如 Qwen/Qwen3-8B）。注意需要支持视觉识别的模型才能用于自动登录验证码识别。

5. 多个模块独立配置 — 自动登录、选课助手、自动评教三个模块的 AI 配置互相独立，你可以为不同模块使用不同的 API Key 和模型。

隐私政策

数据收集与使用

本插件不收集、不存储、不上传任何用户个人数据至开发者服务器。所有配置信息（包括学号、密码、API Key、课表数据、课程评价等）均仅存储于您本地浏览器的 Chrome Storage 中，开发者无法访问这些数据。

第三方 AI 服务

当您使用自动登录（验证码识别）、选课助手等 AI 功能时，插件会将相关请求数据（如验证码图片、课程信息等）发送至您自行配置的第三方 AI 服务提供商（如 SiliconFlow、OpenAI、智谱 AI 等）。这些请求直接由您的浏览器发送至对应 API 地址，不经过开发者服务器。请参阅对应服务提供商的隐私政策以了解其数据处理方式。

权限使用说明

storage — 用于在本地保存您的配置信息、课表数据和课程评价。

activeTab / scripting — 用于在当前标签页注入功能增强脚本（GPA查询、课表抓取、自动评教等），仅在您主动触发的页面上执行。

host_permissions — 用于访问南大相关系统（统一认证、教务系统、LMS、SEEC 等）以提供自动登录、课表同步、LMS 增强等核心功能，以及访问您配置的 AI API 地址以提供 AI 辅助功能。

您的权利

您可以随时在插件设置页面查看、修改或清除所有存储的数据。卸载插件将自动清除所有本地存储数据。如对隐私政策有任何疑问，请通过下方联系方式与开发者联系。

政策更新

本隐私政策可能随插件功能更新而调整，最新版本将在插件设置页面中同步更新。建议您定期查看。
